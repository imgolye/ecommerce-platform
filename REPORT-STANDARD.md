# 汇报标准要求

## ✅ 强制要求

**每次汇报必须包含：**

1. **开发内容** ✅
   - 当前正在开发的模块
   - 新增的文件和代码
   - 实现的功能

2. **进度数据** ✅
   - Phase完成度（百分比）
   - 文件数量
   - 代码行数
   - API接口数量

3. **具体工作** ✅
   - 完成的任务清单
   - 代码位置
   - 功能说明

---

```
## 📊 [时间] 定时汇报

### 🎯 当前进度
- Phase X: XX%

- 新增文件: X个

- 总文件: X个

- 总API: X个

- 时间: XX:XX
```
